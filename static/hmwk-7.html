A Random-Walk Simulation of Server Security

Blog about statistics homeworks

Simulation scope

The scope of this simulation is as follows: given a server that receives weekly security updates for n weeks and m attackers who can breach the system with probability p, we generate random trajectories of the server’s cumulative security score. A value of +1 is assigned for each week in which the server remains secure, and −1 for each week in which a breach occurs. Afterward, the results are compared with a random walk, examining how many trajectories reach each possible total score and how these frequencies converge toward the binomial distribution as n and m increase.

Mathematics behind the simulation

The probability that at least one attacker breaches the server in a given week is calculated as \(1 - (1 - p)^m\), where \(p\) is the probability of a single attacker succeeding and \(m\) is the number of attackers. Therefore, the probability that the server remains secure in a week is \((1 - p)^m\). Over \(n\) weeks, the server's cumulative security score follows a binomial distribution, where each week contributes +1 for security and -1 for a breach.
